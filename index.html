💻 Terminal Basics

🌱 Git Fundamentals

⚙️ First-Time Setup

git config --global user.name "Your Name"
git config --global user.email "your@email.com"
git config --global init.defaultBranch main

📂 Starting a Project

git init   # Initialize new repository

🔄 Common Workflow

git add .                      # Stage changes
git commit -m "message"        # Commit
git remote add origin repo_url # Connect to GitHub
git push -u origin main        # Push

📊 File Stages in Git

  • Untracked → Not in Git
  • Staged → Added with git add
  • Committed → Saved locally
  • Pushed → Uploaded to GitHub

🌿 Branching

git branch              # List branches
git checkout branch     # Switch branch
git checkout -b new     # Create + switch
git branch -d branch    # Delete branch
git merge branch        # Merge branch

♻️ Undoing Changes

Unstage file

git reset file_name
git reset   # Unstage all

Undo last commit

git reset HEAD~1

Hard reset (⚠️ danger)

git reset --hard commit_id

🔗 Git vs GitHub

  • Git → Local version control system 🖥
  • GitHub → Cloud hosting platform ☁️

🚀 Complete Workflow Example

🔧 First Time Setup

git config --global user.name "Your Name"
git config --global user.email "your@email.com"

🆕 New Project

git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/username/repo.git
git push -u origin main

📥 Existing Project

git clone https://github.com/username/repo.git
cd repo

💡 Pro Tips

git commit -am "msg"    # Add & commit tracked files in one step
git log --oneline       # Compact history
git branch -m old new   # Rename branch
git stash               # Temporarily save changes

⚡ Advanced Git Concepts

🚀 Squash Commits (Clean History)

git rebase -i HEAD~5   # Combine last 5 commits
# mark commits as "squash" or "fixup"

📌 Useful before merging PRs to keep history clean.

🎯 Cherry-Pick (Pick a Commit from Another Branch)

git checkout feature
git cherry-pick abc123

✅ Apply a specific bugfix commit without merging the whole branch.

🏷 Tags & Semantic Versioning

git tag -a v1.2.0 -m "Release v1.2.0"
git push origin v1.2.0

🚀 Commonly used for production releases & CI/CD pipelines.

📦 Stash Advanced

git stash push -m "WIP: feature-x"
git stash list
git stash apply stash@{1}

💡 Save unfinished work temporarily without committing.

🔍 Search in Git History

git log -S "functionName"   # Find commits where code was added/removed
git grep "keyword"          # Search keyword in repo files

🧹 Clean Up Local Branches

git fetch -p    # Prune deleted remote branches
git branch -vv  # See tracking branches
git branch -d old-branch

⚡ GitHub CLI Power

gh repo fork repo-url        # Fork a repo
gh pr create --web           # Open PR in browser
gh pr merge 123 --squash     # Merge PR #123 with squash
